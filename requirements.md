# お下がり交換サイト 要件定義

## 概要

知り合い同士（最大30名）限定のお下がり交換サービス。
実際の受け渡し交渉は外部（LINE等）で行う。

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| Frontend | SvelteKit（SPA mode） |
| Database | SurrealDB Cloud |
| Storage | Cloudflare R2（画像） |
| Hosting | Cloudflare Pages |
| Adapter | @sveltejs/adapter-cloudflare |

## ユーザー管理

| 項目 | 内容 |
|------|------|
| 登録方法 | 管理者が直接アカウントを作成し、LINEでID・PINを通知 |
| ユーザーID | 連番の整数（1, 2, 3...）。ログインキーとして使用 |
| 氏名 | 姓・名を別フィールドで登録（表示用）。ひらがな |
| パスワード | 数字4桁PIN |
| ロール | `user` / `admin` |
| 個人情報 | 最小化（メールアドレス等なし） |
| 同姓同名 | IDが異なるため問題なし |

## 出品機能

| 項目 | 内容 |
|------|------|
| 画像 | 複数枚アップロード可（Cloudflare R2） |
| テキスト | 紹介文（自由記述） |
| カテゴリ | 初期なし（将来追加予定） |
| 編集 | 出品者本人が編集可能 |
| 削除 | 出品者本人が削除可能 |

## ステータス管理

```
募集中 ──[ほしいボタン押下(先着1名)]──▶ 交渉中
                                          │
                        ┌─────────────────┤
                        │                 │
              [出品者: 譲渡成立]   [出品者: 交渉決裂]
                        │                 │
                        ▼                 ▼
                      非表示           募集中
```

- 「ほしい」ボタンは `募集中` のときのみ表示
- 交渉中は先着1名のみ（他のユーザーにはボタン非表示または無効化）
- 出品者自身は自分の出品に「ほしい」できない
- ステータス変更（譲渡成立・交渉決裂）は出品者が操作

## 管理者機能

| 機能 | 内容 |
|------|------|
| ユーザー管理 | 登録（ID採番・アカウント作成）・削除・ロール変更 |
| 投稿管理 | 不適切投稿の強制削除 |

## 通知

| フェーズ | 方法 |
|----------|------|
| 初期 | SurrealDB Live Query（ページ閲覧中のみリアルタイム更新） |
| 将来 | Web Push通知（バックグラウンド通知、Service Worker追加） |

## 画面一覧

| 画面 | パス | 権限 |
|------|------|------|
| 出品一覧（トップ） | `/` | ログイン必須 |
| ログイン | `/login` | 全員 |
| 出品作成 | `/items/new` | ログイン必須 |
| 出品編集 | `/items/[id]/edit` | 出品者本人 |
| 管理画面 | `/admin` | admin only |

## 非機能要件

- **セキュリティ**: 管理者のみアカウント作成可能。4桁PINのためブルートフォース対策としてレート制限を実装
- **スケール**: 最大30名、数十〜数百件の出品を想定（SurrealDB Cloudの無料枠で十分）
- **将来拡張**: カテゴリ・サイズ等の属性追加、Push通知対応
